# 🏝 Ostrave Bot — заявки на вечеринку «Остров»

Telegram-бот, в котором клиент **оставляет заявку прямо в чате** за минуту, а
менеджеры **работают с клиентами не выходя из Telegram**: видят заявки, меняют
статусы и пишут клиентам в один тап.

## Что умеет

**Клиент** жмёт «📝 Оставить заявку» → бот по шагам спрашивает имя → удобный
способ связи (Telegram / телефон / WhatsApp) → вариант билета → показывает сводку
и просит подтвердить. Mini App-афиша остаётся опциональной кнопкой, если задан
`WEBAPP_URL`.

**Менеджеры** получают каждую заявку в свой чат с кнопками управления:

| Действие | Как |
|----------|-----|
| Сменить статус | inline-кнопки под заявкой: ⏳ В работе · 💰 Оплачено · 🎉 Завершена · ❌ Отказ |
| Написать клиенту | кнопка «✉️ Написать клиенту» (открывает чат с клиентом) |
| Список заявок | команда `/leads` — последние заявки + сводка по статусам |
| Карточка заявки | команда `/lead <номер>` |

Все заявки хранятся в локальной БД (`leads.db`, SQLite) — это источник правды,
поэтому менеджерские функции работают **без всякой внешней настройки**.
При желании заявки дополнительно зеркалируются в Google-таблицу (статусы тоже
синхронизируются).

## Как это работает

```
Клиент:  /start ──► «📝 Оставить заявку» ──► диалог (имя/связь/билет) ──► подтверждение
                                                                          │
              (опционально) Mini App webapp/ ──► sendData(JSON) ──────────┤
                                                                          ▼
                                              services/leads.submit_application
                                                  │           │            │
                                              storage     чат менеджеров  Google Sheets
                                              (SQLite)    (+ кнопки)       (зеркало)

Менеджер: кнопки статуса / /leads / /lead <id>  ──► storage + Google Sheets
```

## Стек
Python 3.11+, [aiogram 3](https://docs.aiogram.dev/), SQLite (stdlib),
gspread (Google Sheets — опционально).

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate          # Windows
pip install -r requirements.txt
```

## Настройка

1. **Бот.** У [@BotFather](https://t.me/BotFather) создайте бота → получите `BOT_TOKEN`.
2. **Чат менеджеров.** Создайте группу, добавьте туда бота. Чтобы узнать
   `MANAGERS_CHAT_ID`, добавьте в группу [@getmyid_bot](https://t.me/getmyid_bot).
   Для супергрупп ID отрицательный. В этом чате работают `/leads`, `/lead` и кнопки.
3. Скопируйте `.env.example` → `.env`, заполните `BOT_TOKEN` и `MANAGERS_CHAT_ID`.
   Этого достаточно для запуска.

### Необязательно

- **Mini App-афиша.** Соберите и захостите `webapp/` на HTTPS (см.
  [`webapp/README.md`](webapp/README.md)), URL положите в `WEBAPP_URL` —
  у клиента появится кнопка «🎟 Открыть афишу». Без `WEBAPP_URL` заявку
  оформляют прямо в чате.
- **Google Sheets.** Включите Sheets API + Drive API, создайте сервис-аккаунт,
  скачайте JSON-ключ → `creds.json`, расшарьте таблицу на `client_email` из ключа,
  заполните `GSHEET_ID`. Без этого заявки всё равно сохраняются в `leads.db`.

## Запуск

```bash
python bot.py
```

Откройте бота, нажмите `/start`, затем «📝 Оставить заявку».

## Структура

| Файл | Назначение |
|------|------------|
| `bot.py` | точка входа, polling, инициализация БД и меню команд |
| `config.py` | загрузка `.env` |
| `keyboards.py` | клавиатуры: меню, контакт, билеты, подтверждение, кнопки менеджера |
| `models.py` | модель `Application`, статусы заявки |
| `handlers/start.py` | `/start`, «О вечеринке» |
| `handlers/application.py` | диалог «оставить заявку» (FSM) |
| `handlers/manager.py` | кнопки статусов, `/leads`, `/lead` |
| `handlers/webapp.py` | приём заявки из Mini App (опционально) |
| `services/storage.py` | хранилище заявок (SQLite) — источник правды |
| `services/leads.py` | оркестрация заявки: сохранить → уведомить → в таблицу |
| `services/sheets.py` | зеркалирование и синхронизация статусов в Google Sheets |
| `webapp/` | Mini App (Vite + React) — опциональная афиша |
