- [x] Карточка менеджера: разбивка «аванс / на острове при выезде» — _2026-06-26_.
  Реальный операционный риск (нашёл сквозной проверкой): для заявок из калькулятора поле
  «💰 Сумма» включает вход на остров + аренду, которые гость платит на острове при выезде,

  (та же групповая цена билета, что калькулятор кладёт в open-air), «на острове» = amount −

  под «Сумма» строки «├ 💳 аванс (перевод): N ₽ / └ 🏝 на острове при выезде: M ₽», только
  когда есть оплата на месте (для прямых заявок сумма = аванс, разбивки нет). Не меняет саму
  «Сумму» — совместимо с любым будущим решением. Проверено: калькулятор 12 чел = 78900 →
  18000 аванс + 60900 на острове; прямая 4 чел = 6000 без разбивки. Сборка OK.

## Notes

- НЕ запускать `python bot.py` напрямую — на том же токене работает прод-экземпляр (TelegramConflictError).
- Проверка сборки: `python -c "import handlers, keyboards, models; handlers.setup_routers(); print('OK')"`.
- **Выкатка в прод:** `powershell -ExecutionPolicy Bypass -File restart_bot.ps1` — делает pre-flight проверку импорта, останавливает старый экземпляр и поднимает один новый. При ошибке импорта старый бот остаётся работать.
- Бот работает в фоне (detached). Логи: `bot.err.log` (основной, aiogram пишет в stderr) и `bot.out.log`; прошлые логи ошибок сохраняются как `bot.err.<timestamp>.log`.
- Остановить бота вручную: найти PID через `Get-CimInstance Win32_Process -Filter "Name='python.exe'" | ? { $_.CommandLine -match 'bot\.py' }` и `Stop-Process -Id <PID> -Force`.
- Не коммитить/не трогать .env, creds.json, bot.err.log/bot.out.log; leads.db менять только миграциями схемы.
